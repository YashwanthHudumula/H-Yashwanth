01 Overview

What This System Does

Modern organizations need both financial transparency and physical security — but most tools handle these separately. This project integrates real-time expense tracking with intelligent facial recognition into a single, unified platform.

The system features role-based access for Admins and Managers, an interactive expense dashboard with charts and filters, and a Python-based face recognition module that monitors inventory entry/exit via CCTV — logging every access with a timestamp.

A survey of 20 businesses showed 90% supported the integration: 85% found current tools inadequate, and 70% reported unauthorized inventory access incidents.

Problem Being Solved
  • → Expense tools have no inventory access logging
  • → CCTV systems require manual footage review
  • → No unified platform for finance + security
  • → Unauthorized access goes undetected
Key Features
  • ✓ Real-time face recognition via CCTV + DeepFace
  • ✓ Expense management with category filters & charts
  • ✓ Timestamped security logs for all entries/exits
  • ✓ Admin/Manager role-based access control
  • ✓ Unknown face flagging with automatic alerts

03 Core Modules

Six Key Modules

🔐
Module 01
User Authentication
Role-based login for Admin and Manager. Secure session management. Unauthorized access redirects to login screen. Dashboard loads based on user role.
💰
Module 02
Expense Management
Add, update, and categorize expenses (purchase, utility, transport). Monthly/yearly filters. Timestamped transactions. Full CRUD with REST APIs tested via Postman.
📹
Module 03
Inventory Surveillance
CCTV-based face detection at entry/exit points using OpenCV. Every detection logged with timestamp. Unknown faces flagged. Real-time Python threading for non-blocking performance.
🧠
Module 04
Face Recognition
DeepFace library for identity verification. Pre-registered users matched against live CCTV feed. MATCH/NO MATCH output displayed. Logs pushed to backend database.
📊
Module 05
Dashboard & Analytics
Interactive line/bar charts for daily expense visualization. Filters by date, category, amount, and quantity. Transaction status management (Completed/Pending/Cancelled).
🔌
Module 06
API & Data Layer
RESTful APIs built with Spring Boot. Postman tested for reliability. MySQL schema: User, Product, Category, Supplier, Transaction tables with proper foreign keys.
