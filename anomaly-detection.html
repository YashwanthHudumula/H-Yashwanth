Overview
Project Overview

Modern building heating systems generate enormous volumes of sensor data, but detecting faults is challenging due to the complete absence of labeled anomaly data. This project presents a fully unsupervised anomaly detection pipeline built around the FP-Growth association rule mining algorithm, following the CRISP-DM methodology.

The pipeline was applied to approximately 1.5 million time-stamped sensor records from a simulated multi-room building heating system with 8 indoor rooms. The system mines static baseline rules from unlabeled training data, then scores anomalies in streaming test transactions — outputting interpretable alerts that facility managers can act on directly.

Co-authored with Venkata Naga Sai Yaswanth Deevi at Blekinge Institute of Technology, Karlskrona, Sweden.

Pipeline
CRISP-DM Pipeline
Phase 01
Data Understanding
~1.5M records across 8 rooms (A11–A24) + outdoor sensor. Features: temperature (K), actuation signal, setpoint, previous 3 actuation values sampled every 10s.
Phase 02
Feature Engineering
Three derived features: temperature delta, actuation delta, and setpoint deviation. Continuous values discretised into symbolic states (TEMP_COLD, HEATER_ON, etc.).
Phase 03
Sliding Window Transactions
Average-based sliding windows (W=6 records, ~60s; stride=3, ~30s). Averaging before discretisation eliminates contradictory labels during state transitions.
Phase 04
FP-Growth Rule Mining
Applied to all training transactions with min_support=0.10 and min_confidence=0.92. Generated 70,672 candidates, filtered to 38 high-confidence rules.
Phase 05
Weighted Anomaly Scoring
Each test transaction scored using frequency-weighted formula considering rule confidence, support, and violated vs. applicable rules. THREE severity tiers: LOW, MEDIUM, HIGH.
Phase 06
Pattern Stability Analysis
Training data split into K=5 chronological folds. Each rule scored for temporal stability. 7 STABLE rules identified that hold across all folds.
Phase 07
Cross-Method Validation
Results compared against Isolation Forest, K-Means, and DBSCAN on a shared 6-dimensional feature space. Pairwise agreement 93–97%.

Cross-Method Comparison

All methods show 93–97% pairwise agreement, confirming consistent identification of normal behaviour. ARM's 2.7% rate falls between Isolation Forest (5%) and DBSCAN (1.1%), providing balanced sensitivity with unique interpretability advantages unavailable from black-box methods.

Key Contributions
Novel Contributions

• Average-based sliding window transaction generation — eliminates contradictory labels during state transitions, improving rule quality significantly over individual-record methods.

• Frequency-weighted anomaly scoring — rare-item violations weighted at 0.4× to reduce false positives from infrequent but normal events.

• Temporal stability analysis across 5 chronological folds — identifies a core invariant rule set that holds across seasonal changes.

• Unified comparison framework — all four methods evaluated under a common 6-dimensional feature space for fair cross-method assessment.
